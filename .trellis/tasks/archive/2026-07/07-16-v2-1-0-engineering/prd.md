# v2.1.0 工程质量与安全加固

## Goal

在 v2.0.4 稳定基线之上降低模块复杂度、配置损坏风险和隐式代码执行风险，并建立可持续质量门禁。

## Requirements

- 子任务按模块边界、持久化、安全、性能、CI、官网媒体、规范文档和发布顺序推进。
- 重构默认保持公开行为与配置兼容；行为变化必须在所属安全任务中明确记录。
- 共享文件和根级配置串行修改，避免多个任务交叉提交。

## Acceptance Criteria

- [x] 10 个子任务完成并各有一个独立提交。
- [x] PowerShell/shell 大文件、非原子写入、无确认变更和无信任 hook 得到治理。
- [x] 50 项目 status 性能目标有可重复基准证据。
- [x] 全量质量门禁通过后统一发布 v2.1.0；GitHub/Scoop 已公开验证，
  PowerShell Gallery 因环境缺少 API key 作为唯一外部阻塞记录。
