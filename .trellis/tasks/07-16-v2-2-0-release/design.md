# v2.2.0 Release Design

## Release Identity

- Version: `2.2.0`; tag: `v2.2.0`.
- Canonical version source: `cdp.psd1` `ModuleVersion`.
- Release commit: the final `main` commit after release evidence and metadata are synchronized and all local gates pass.
- Asset: `artifacts/release/cdp-2.2.0.tar.gz`, generated by `scripts/New-ScoopPackage.sh` and retained unchanged between local hash verification and GitHub upload.
- Scoop identity: the asset URL, extract directory and exact SHA-256 in `scoop/cdp.json`.

## Preflight Audit

Verify the five automation child tasks are archived, their work commits are reachable from `main`, and their completion evidence covers PowerShell and shell behavior. Confirm the release task is the only unfinished automation child.

Audit `cdp.psd1`, `src/cdp.psm1`, `src/cdp.sh`, `install-wsl.sh`, `tests/cdp.Tests.ps1`, `CHANGELOG.md`, `PROGRESS.md`, `scoop/cdp.json`, `README.md`, and `README_ZH.md` as one release contract. Explicitly check compatibility and migration notes for status schema version 1, optional platform paths, stable raw-path workspace references, exec safety, and frecency state.

Read-only remote checks compare local `main`, `origin/main`, existing tags and GitHub Releases before any remote mutation. Unexpected divergence or an existing `v2.2.0` identity stops publication.

## Local Quality Matrix

1. PowerShell 7 unified quality gate, including Pester, coverage, PSScriptAnalyzer and release metadata. Windows PowerShell 5.1 runs the identical gate in hosted CI.
2. Generated shell check, syntax, ShellCheck, modularization, CLI/status/status-JSON/path/frecency/workspace/exec/performance/safe-mutation/v2/persistence/installer regressions under bash and zsh, plus the pinned Bash 3.2 image.
3. Documentation structure, exported-command coverage, Web asset/media budgets and Playwright Chromium smoke.
4. Deterministic Scoop package contents/hash, release metadata, YAML/JSON parsing and `git diff --check`.
5. Fixed status benchmark, with results recorded as evidence but judged against regression thresholds owned by the benchmark tests.

The retained archive is generated only after source and metadata gates pass. Record its byte size, SHA-256 and content list; a second temporary build must produce the same digest.

## Remote Release Sequence

1. Reconfirm a clean worktree and remote identity, then `git pull --rebase --autostash` and push the final release commit.
2. Watch the exact `main` CI run for that commit until all jobs succeed.
3. Create annotated `v2.2.0` at the verified release commit, push it, and compare local/remote peeled tag SHAs.
4. Create a public, non-prerelease GitHub Release from the verified tag and upload the retained archive.
5. Download the public asset and compare its SHA-256 and size with the retained archive and Scoop manifest.
6. Publish PowerShell Gallery only when `PS_GALLERY_API_KEY` is present; verify `Find-Module` and the exact version page after indexing.
7. Verify the Scoop URL resolves to the same archive and run isolated source/package installation smoke where the host supports it.

## State And Compatibility

v2.2.0 does not require eager migration. Existing `rootPath` configs remain valid, platform mappings are optional, existing workspace names migrate to stable raw-path references through existing lifecycle commands, and recent state continues to use raw `rootPath` identity. Downgrade preserves unknown metadata because JSON mutations retain unknown fields; older runtimes ignore the optional v2.2.0 fields but cannot consume commands introduced by v2.2.0.

## Rollback

- Before tag/Release: fix the defect, recommit, rerun the affected and full gates, then use the new commit as release identity.
- After tag but before public Release/Gallery: move the annotated tag only after the replacement commit is pushed and revalidated.
- After a public channel publishes: do not replace immutable artifacts silently; prepare a patch release.
- Missing Gallery credentials are an external channel blocker and must remain explicitly visible without blocking truthful completion of GitHub/Scoop publication evidence.
