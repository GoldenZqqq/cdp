# Product

## Register

brand

## Users

cdp 面向同时维护多个 Git 仓库、频繁使用终端，并在 Claude Code、Codex、Gemini CLI、Cursor、VS Code、PowerShell、WSL、Linux 或 macOS 之间切换工作上下文的开发者。

他们真正要完成的工作不是“切换目录”，而是在最短时间内找到正确项目、确认仓库状态，并进入可以立即编码的 AI CLI 或编辑器上下文。

## Product Purpose

cdp 把长路径导航、多仓库 Git 状态检查、项目配置管理和 AI CLI 启动压缩成少量可记忆命令。官网的目标是在十秒内让新访客理解这一价值，在一分钟内看到真实工作流，并能无摩擦地选择平台完成安装。

成功意味着访客不再把 cdp 理解为另一个 `cd` 别名，而把它记住为面向 Vibe Coding 工作流的终端项目工作台。

## Brand Personality

迅疾、可靠、锋利。

品牌语气像清晰的高速公路导向标识：短句、确定、没有营销废话。视觉可以有速度感和技术张力，但始终保持工程可信度，不扮演黑客电影中的终端道具。

## Anti-references

- 泛用深色 SaaS 模板：渐变 hero、玻璃卡片和一排排同构功能卡。
- 廉价赛博朋克：满屏霓虹、故障闪烁、不可读的装饰性终端字符。
- “AI 提效”空话：只声称更快，却不展示真实命令、状态和结果。
- 终端角色扮演：为了显得技术而堆叠无意义代码、网格背景和单色 mono 字体。
- 文档复制站：把 README 原样搬到网页，缺乏清晰叙事和视觉节奏。

## Design Principles

1. **先击中损耗，再展示功能。** 先让开发者认出长路径、多仓库遗忘和 AI CLI 上下文重载的痛，再给出 cdp 的压缩方案。
2. **展示，不宣称。** 每个核心价值都由真实命令、状态或演示视频证明。
3. **一条命令就是一条路径。** 页面叙事围绕输入、路由、到达展开，保持品牌动作一致。
4. **像路标一样清楚。** 层级、文案和交互必须在快速扫视中可理解，不依赖装饰解释。
5. **速度必须是真的。** 静态、轻量、渐进增强；官网自身不能用迟缓体验宣传效率工具。

## Accessibility & Inclusion

目标为 WCAG 2.2 AA。正文和交互文本满足对比度要求，所有功能支持键盘访问和清晰焦点状态，状态信息不只依赖颜色表达。动画尊重 `prefers-reduced-motion`，页面在禁用 JavaScript 时仍可阅读并完成安装。中文和英文内容保持同等完整，语言切换会同步更新文档语言属性与可访问名称。
